# Vauldy 桌面端客户端

> 需求规格说明书（SRS）

---

## 1. 项目概述

### 1.1 项目背景

Vauldy 是一款自托管的家庭数字保险箱，以 Go 后端 + React Web 前端提供影视、音乐、照片、文档等媒体的安全存储与在线浏览能力。Web 端虽功能完整，但在桌面环境下用户期望获得更接近原生应用的体验：独立窗口、系统托盘、快捷键、高性能本地播放器、多窗口与文件关联等。

为满足桌面用户场景，需开发 **Vauldy 桌面端客户端**（以下简称「桌面客户端」），在 Windows、Linux、macOS 上提供统一体验，并复用 Vauldy Media Server 现有 API，无需改动后端核心协议。

### 1.2 项目目标

基于 **Tauri 2** + **mpv** 技术栈，开发 Vauldy 桌面端客户端，实现：

- 对接 **Vauldy Media Server** REST API（`/api/v1`）及流媒体端点
- 一套代码构建 **Windows / Linux / macOS** 三平台安装包
- 使用 **mpv** 作为核心播放引擎，支持视频、音乐的高性能播放（HLS、直链、多音轨、字幕）
- 支持媒体库浏览及 **视频、音乐、图片、文档** 在线预览
- UI 风格与 **Vauldy Web** 保持一致，降低用户学习成本
- 独立代码仓库维护（`knoxmedia/Vauldy-Desktop`），通过 Git Submodule 嵌入主项目便于联调

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| Vauldy Media Server | 自托管媒体服务端，默认监听 `8200` 端口 |
| 桌面客户端 | 本项目的 Tauri 应用，运行于 Windows / Linux / macOS |
| Tauri | 基于 Rust 的轻量跨平台桌面框架，WebView 渲染 UI，Rust 处理系统能力 |
| mpv | 开源高性能媒体播放器，通过 IPC/嵌入方式由 Tauri 调用 |
| 播放计划（Playback Plan） | `GET /media/:id/hls` 返回的 JSON，描述播放方式与 URL |
| access_token | JWT；流媒体 URL 可通过 `?access_token=` 鉴权 |
| Sidecar | Tauri 将 mpv 可执行文件打包为附属进程，由 Rust 启动并通信 |

### 1.4 项目范围

**包含：**

- 终端用户日常消费功能（浏览、播放、收藏、搜索、续播等）
- 三平台安装包（`.msi` / `.deb` / `.AppImage` / `.dmg` 等）
- mpv 视频/音乐播放、图片查看、文档阅读

**不包含（首期）：**

- 管理后台（`adm` 路由：库管理、扫描、转码、用户管理等）
- 本地上传媒体至服务器
- 完整离线媒体缓存（可作二期）
- 演职人员模块（待后端 API 就绪）

│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐ │
│  │  Web UI         │  │  Tauri Commands │  │  mpv Player     │ │
│  │  (React + TS)   │◄─┤  (Rust)         │──►│  (Sidecar/IPC)  │ │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘ │
│           │                    │                      │           │
│           │           系统托盘 / 快捷键 / 窗口管理    │           │
└───────────┼────────────────────┼──────────────────────┼───────────┘

### 3.2 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 桌面壳 | Tauri 2.x | 体积小、安全、跨平台；Rust 后端 + 系统 WebView |
| 前端 UI | React 19 + TypeScript + Vite | 与 Vauldy Web 技术栈对齐，可复用组件与 API 客户端逻辑 |
| 样式 | CSS Modules / 复用 Web 主题变量 | 深色主题、`#0f1419` / `#00a4dc` / `#ed6d00` |
| 状态 | Zustand | 认证、播放器、设置 |
| HTTP | Axios | API 客户端，对齐 `web/src/api/client.ts` |
| 视频/音乐播放 | **mpv** | Sidecar 或 libmpv 嵌入；HLS、字幕、多音轨 |
| 图片 | WebView / 原生图像组件 | 缩略图网格 + 全屏查看 |
| PDF | pdf.js（WebView） | 与服务端预览 URL 或本地渲染 |
| EPUB | epub.js（WebView） | 与 Web 端一致 |
| Office 文档 | 服务端转 PDF + pdf.js | `GET /document/preview.pdf` |
| 国际化 | i18next | zh-CN、zh-TW、en、ja、ko |
| 打包 | Tauri Bundler | Windows NSIS/MSI、Linux deb/AppImage、macOS dmg |

### 3.3 mpv 集成方案

| 方案 | 说明 | 推荐阶段 |
|------|------|----------|
| **Sidecar** | 打包 mpv 可执行文件，Tauri 启动子进程，通过 `--input-ipc-server` JSON IPC 控制 | 一期 MVP |
| **嵌入窗口** | `--wid` 将 mpv 视频输出嵌入 Tauri 窗口指定区域（Windows X11/macOS） | 一期 |
| libmpv | Rust 绑定 libmpv，深度集成 | 二期（可选） |

**一期播放流程：**

1. 前端请求 `GET /media/:id/hls` 获取播放计划
2. Rust 命令根据计划构造 mpv 启动参数（`--url=...` 或 HLS master）
3. URL 附加 `access_token` 查询参数
4. mpv Sidecar 启动，视频区域嵌入主窗口或独立播放窗口
5. 前端通过 IPC 发送 pause/seek/volume 等命令；mpv 事件回传进度

### 4.4 视频播放（mpv）

#### 4.4.1 播放流程

```
用户点击播放
    ▼
GET /api/v1/media/:id/hls  → 播放计划
    ▼
构造 mpv 命令行 / IPC
    ├── hls_master     → mpv 播放 HLS（--demuxer-lavf-o=protocol_whitelist=...）
    ├── fallback/play  → 直链 MP4/MKV
    └── JIT session    → HLS master URL
    ▼
URL 附加 ?access_token=
    ▼
mpv 嵌入窗口或独立播放窗
    ▼
进度上报 POST /media/:id/progress
```

#### 4.4.2 mpv 能力（一期）

| 功能 | mpv 实现 |
|------|----------|
| HLS 自适应 | 原生 demuxer |
| 直链 progressive | 直接 `--` URL |
| 字幕 | `--sub-file` 或加载 `/subtitles/:sid/vtt` |
| 多音轨 | `--audio-file` / `cycle audio` |
| 倍速 | `set speed` |
| 全屏 | `fullscreen` 属性 |
| 预览缩略图 | `--osd-bar` + 外部 sprite（二期） |
| 断点续播 | `--start=秒数` |

#### 4.4.3 DRM 说明

| DRM | 桌面端支持 |
|-----|------------|
| HLS AES-128 | mpv 可通过 `--demuxer-lavf-o` 或自定义 key loader（一期可行） |
| PowerDRM | 需专用解密逻辑，参考 Web PowerPlayer（二期） |
| Widevine / FairPlay | 桌面 mpv 支持有限，优先非 DRM 库或 HLS AES-128（二期评估） |

一期优先支持 **非 DRM 库** 与 **HLS 预转码 / 直链** 播放；DRM 库提示用户使用 Web 端或标注「暂不支持」。

## 5. 接口集成规范

### 5.1 认证

- JSON API：`Authorization: Bearer <JWT>`
- 流媒体 / 图片：`?access_token=<JWT>`（mpv、`<img>` 无法设 Header）

### 5.2 URL 构建

复用 Web 端 `withAccessToken`、`mediaPosterSrc`、`mediaPlaySrc` 逻辑（`web/src/api/client.ts`）。

### 5.3 错误处理

| 状态 | 行为 |
|------|------|
| 401 | 清除 Token，跳转登录 |
| 403 | 权限提示（含家长控制） |
| 网络错误 | 重试 + 离线提示 |

## 6. 非功能需求

### 6.1 性能

| 指标 | 目标 |
|------|------|
| 冷启动 | ≤ 2s（已登录） |
| 首页首屏 | ≤ 1.5s（局域网） |
| 视频起播 | ≤ 3s（HLS/直链） |
| 安装包体积 | ≤ 80 MB（含 mpv sidecar，视平台而定） |
| 内存占用 | 空闲 ≤ 150 MB；播放 ≤ 400 MB |

### 6.2 兼容性

| 平台 | 最低版本 |
|------|----------|
| Windows | 10 1809+（x64） |
| Linux | Ubuntu 20.04+ / 主流发行版 x64 |
| macOS | 11 Big Sur+（Intel / Apple Silicon） |

### 6.3 安全

- Tauri CSP 限制 WebView 仅访问配置的服务器域
- Token 存系统凭据管理器（Windows Credential Manager / macOS Keychain / libsecret）
- 禁止在日志中输出 Token
- 自动更新签名验证（Tauri Updater，二期）

### 6.4 可用性

- 深色主题默认，与 Web 一致
- 窗口大小记忆、多显示器支持
- 高 DPI 适配
- 基础无障碍（键盘导航）
